# Contributing

Thanks for wanting to contribute to this repository — contributions are welcome!

Please follow the guidelines below to make the process smooth for everyone.

**Code of Conduct**

- Read and follow the [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).

**How to contribute**

- **Issues**
	- Search existing issues before opening a new one.
	- Use a clear title and include steps to reproduce, expected behavior, and actual behavior.
	- Provide relevant environment details (OS, Dart SDK version) when reporting bugs.

- **Pull Requests**
	1. Fork the repository and create a branch named `fix/short-description` or `feat/short-description`.
	2. Write clear, focused commits. Rebase or squash before opening the PR if appropriate.
	3. Link related issues in the PR description and explain the change and rationale.
	4. Keep changes small and focused; add tests for bug fixes and new features when applicable.
	5. Ensure code is formatted consistently and that any provided examples or demos still work.
	6. Open the PR against the `main` branch and add reviewers.

**Coding style & tests**

- Follow Dart style conventions. Run `dart format` where applicable.
- Add or update tests when introducing fixes or features.

**Documentation**

- Update `README.md`, relevant lesson files under `Pertemuan X/`, or other docs when behavior or examples change.

**Security & Sensitive Issues**

- For security vulnerabilities, do not open a public issue. See [SECURITY.md](SECURITY.md) for responsible disclosure instructions.

**License & Attribution**

- By contributing, you agree that your contributions will be licensed under this repository's license (see `LICENSE`).

**Questions or help**

- If you're unsure where to start, open an issue describing your idea and maintainers will help.

Thank you for your contributions!
